# @synkro/core v0.15.1

**Release date:** 2026-03-09

## Changes

### FT-10: Custom transport support

`SynkroOptions.transport` now accepts a `TransportManager` instance in addition to `"redis"` and `"in-memory"` strings. This enables custom transport implementations (e.g., HTTP-based dispatch for serverless environments) without modifying the core package.

```typescript
import { Synkro, type TransportManager } from "@synkro/core";

const customTransport: TransportManager = { /* ... */ };

const synkro = new Synkro({
  transport: customTransport,
  // ...
});
```

### FT-11: Standalone `executeHandler()` utility

Extracted the handler execution logic (dedup, distributed locking, retry with backoff, metrics, completion event publication) from `HandlerRegistry` into a standalone `executeHandler()` function. This allows external packages to run handlers with the same guarantees as core's internal processing.

```typescript
import { executeHandler } from "@synkro/core";

const result = await executeHandler({
  transport,
  channel: "user.created",
  requestId: "req-123",
  payload: { email: "alice@example.com" },
  handler: async (ctx) => { /* ... */ },
  retention: { lockTtl: 30, dedupTtl: 3600 },
});
// result: { status: "completed" | "skipped_dedup" | "skipped_lock" | "failed", error? }
```

Both `ExecuteHandlerOptions` and `ExecuteHandlerResult` types are exported.
