# @synkro/core v0.10.0

**Release date:** 2026-03-07

## Changes

### TD-01: Retry backoff strategy

`RetryConfig` now supports configurable delay and backoff strategies for handler retries, eliminating thundering-herd behavior under failure conditions.

**New `RetryConfig` options:**

- `delayMs` — Base delay between retries in milliseconds (default: `1000`)
- `backoff` — `"fixed"` (default) or `"exponential"` (delay doubles each attempt)
- `jitter` — When `true`, randomizes the delay within ±50% of the computed value to spread retry storms
- `retryable` — Predicate `(error: unknown) => boolean` to skip retries for non-retryable errors

**Example:**

```ts
synkro.on("order:process", handler, {
  maxRetries: 3,
  delayMs: 500,
  backoff: "exponential",
  jitter: true,
  retryable: (err) => !(err instanceof ValidationError),
});
```

This produces delays of approximately 500ms, 1000ms, 2000ms (±50% jitter), and skips retries entirely for `ValidationError`.

**New exported type:** `RetryBackoffStrategy` (`"fixed" | "exponential"`)
